# 「現場ポケット勤怠管理」ランディングページ コーディング仕様書

## 1. プロジェクト概要

### 1.1 目的
「現場ポケット」の勤怠管理機能を紹介するランディングページ。2024年4月からの労働時間管理に対応した製品の特徴、機能、価格などを訴求し、無料デモの申し込みにつなげることを目的としています。

### 1.2 ターゲットユーザー
- 建設業・工事業の経営者・管理者
- 勤怠管理や給与計算に課題を抱えている方
- 2024年4月の労働基準法改正対応に不安を持つ方

## 2. 技術スタック

### 2.1 フロントエンド技術
- HTML5
- CSS3 (Flexbox, Grid, CSS Variables)
- JavaScript (バニラJS)
- FontAwesome 6.4.0 (アイコン表示)
- Google Fonts (Noto Sans JP)

### 2.2 開発環境
- 静的ウェブサイト

## 4. ページ構成・セクション分析

### 4.1 全体構造
ランディングページは以下のセクションで構成されています：

1. ヒーローセクション (hero)
2. 問題点セクション (problems)
3. 法改正セクション (legal-change)
4. 機能紹介セクション (features)
5. 選ぶ理由セクション (reasons)
6. 価格セクション (pricing)
7. お問い合わせセクション (contact)
8. フッター (footer)

### 4.2 各セクション詳細

#### 4.2.1 ヒーローセクション
- クラス: `hero`, `bg-image`
- 背景: グラデーション + 背景画像 (`img/mv.jpg`)
- コンポーネント:
  - サブタイトル (`hero-subtitle`)
  - メインタイトル (`hero-title`)
  - 説明文 (`hero-description`)
  - CTAボタン (`hero-buttons`内の`cta-button`)

#### 4.2.2 問題点セクション
- ID: `problems`
- グリッドレイアウト (3列×2行)
- コンポーネント:
  - セクションタイトル (`section-title`)
  - 問題カード (`problem-card`) x 6枚
    - 画像 (`img`)
    - アイコン (`problem-icon`)
    - 問題タイトル (`problem-title`)
    - 問題の説明 (`problem-description`)

#### 4.2.3 法改正セクション
- ID: `legal-change`
- 2カラムレイアウト (画像 + テキスト)
- コンポーネント:
  - セクションタイトル (`section-title`)
  - 法改正内容の画像 (`legal-image`)
  - テキストコンテンツ (`legal-text`)
  - 法律ポイント (`legal-point`) x 2つ
  - 質問コンテナ (`legal-question-container`)

#### 4.2.4 機能紹介セクション
- ID: `features`
- グリッドレイアウト (3列×2行)
- コンポーネント:
  - セクションタイトル (`section-title`)
  - セクションサブタイトル (`section-subtitle`)
  - 機能カード (`feature-card`) x 6枚
    - 画像 (`img`)
    - アイコン (`feature-icon`)
    - 機能タイトル (`feature-title`)
    - 機能説明 (`feature-description`)
  - CTAボタン (`cta-buttons`)

#### 4.2.5 選ぶ理由セクション
- ID: `reasons`
- 3カードレイアウト
- コンポーネント:
  - セクションタイトル (`section-title`)
  - 理由カード (`reason-card`) x 3枚
    - 画像 (`img`)
    - アイコン (`reason-icon`)
    - タイトル (`reason-title`)
    - 説明 (`reason-description`)

#### 4.2.6 価格セクション
- ID: `pricing`
- コンポーネント:
  - セクションタイトル (`section-title`)
  - セクションサブタイトル (`section-subtitle`)
  - 価格カード (`pricing-card`)

#### 4.2.7 お問い合わせセクション
- ID: `contact`
- コンポーネント:
  - セクションタイトル (`section-title`)
  - セクションサブタイトル (`section-subtitle`)
  - CTAボタン (`cta-buttons`)

#### 4.2.8 フッター
- クラス: `footer`
- 固定フッター（PC表示時）
- コンポーネント:
  - ロゴ (`logo`)
  - ナビゲーションリンク (`nav-links`)
  - CTAボタン (`cta-button`)

## 5. レスポンシブデザイン

### 5.1 ブレイクポイント
- PC: 769px以上
- タブレット: 481px 〜 768px
- スマートフォン: 480px以下

### 5.2 主要なレスポンシブ対応

#### 5.2.1 問題点セクション
- PC: 3列グリッド
- タブレット: 2列グリッド
- スマホ: 1列（フル幅）

#### 5.2.2 機能紹介セクション
- PC: 3列グリッド
- タブレット: 2列グリッド
- スマホ: 1列（フル幅）

#### 5.2.3 選ぶ理由セクション
- PC: 3列横並び
- タブレット: 3列（フォントサイズ調整）
- スマホ: 1列（フル幅）

#### 5.2.4 フッター
- PC: 固定フッター（画面下部に固定）
  - `position: fixed` で画面下部に固定
  - `z-index: 1000` で最前面に表示
  - メインコンテンツに `padding-bottom` を追加してフッターと重ならないように調整
- スマホ: 通常フロー（スクロールに追従）
  - 固定を解除して通常のフローに戻す
  - メインコンテンツの余白も削除

### 5.3 ボタンの表示/非表示
1. **ヒーローセクションのボタン**
   - 「実際の画面を見る」ボタン（`.screen-preview`）
     - スマホ表示: 表示（青色ボタン、幅調整あり）
     - PC/タブレット: 青色グラデーションで表示
     - サイズ: スマホ時は `width: 208px`, `height: 54px`, `font-size: 2rem`
   - 「今すぐ始める」ボタン（`.cta-button:not(.screen-preview)`）
     - スマホ表示: 非表示（`display: none !important`）
     - PC/タブレット: オレンジ色のグラデーションボタンで表示

2. **フッターのCTAボタン**
   - スマホ表示: 非表示（`display: none`）
   - PC/タブレット: 表示

3. **ボタンのスタイル**
   - スマホ表示時はボタンの幅を `100%` に拡張（最大幅 `280px`）
   - ボタンの高さ: スマホ時は `56px`、PC/タブレット時は `69px`
   - フォントサイズ: スマホ時は `1.8rem`、PC/タブレット時は `2.4rem`

## 6. レスポンシブデザイン対応

メディアクエリを使用して複数の画面サイズに対応：
- モバイル: 480px以下
- タブレット: 768px以下
- 通常デスクトップ: 769px以上

主な調整ポイント:
- フォントサイズの縮小
- グリッドレイアウトの列数調整
- 余白（マージン・パディング）の調整

## 7. JavaScript機能

### 7.1 実装機能
- スクロールアニメーション（フェードイン効果）
- スムーズスクロール
- モーダルウィンドウ機能
  - モーダルの表示/非表示の制御
  - 背景スクロールの無効化/有効化
  - モーダル外クリックでの閉じる機能
  - ESCキーでの閉じる機能

### 7.2 主要関数
- `checkFade()`: 要素が画面内に入った時にフェードイン効果を適用
- イベントリスナー
  - `scroll`: スクロール検知と要素のフェードイン制御
